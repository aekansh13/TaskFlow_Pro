How to Use This Prompt Guide
This document contains 18 ready-to-paste prompts organized in the exact order you should build the project. Each prompt is self-contained — it gives the AI tool everything it needs to produce working, integrated code.
Golden Rules for Vibe Coding
Always run each prompt in sequence — later prompts build on files created by earlier ones.
After each prompt, read the generated code. Do not skip this. Your faculty WILL ask you to explain it.
If the AI produces code that doesn't match your folder structure, paste your current file tree and say: 'Adjust all import paths to match this structure.'
Keep a single chat session per Phase. This gives the AI memory of what it already built.
If something breaks, paste the full error into the chat and say: 'Fix this error. Do not change anything else.'
After each phase is complete, commit to GitHub with a meaningful message like: 'Phase 2 complete: auth endpoints working'.
Which AI tool should I use? All prompts work with Claude (claude.ai), ChatGPT, Gemini, GitHub Copilot Chat, and Cursor. For file-editing tools like Cursor and Copilot, open the relevant file first, then paste the prompt. For chat-based tools like Claude and ChatGPT, use a fresh session per phase.
Prompt Variable Convention: Some prompts contain placeholders in [SQUARE BRACKETS]. Replace these with your actual values before sending. For example, [YOUR_MONGO_URI] should be replaced with your real MongoDB connection string.

CREATE: backend/celery_app.py
Initialize Celery using the Flask app factory:
def make_celery(app):
    celery = Celery(app.import_name, broker=app.config['CELERY_BROKER_URL'])
    celery.conf.update(app.config)
    class ContextTask(celery.Task):
        def __call__(self, *args, **kwargs):
            with app.app_context():
                return self.run(*args, **kwargs)
    celery.Task = ContextTask
    return celery
CELERYBEAT_SCHEDULE = {
    'auto-repeat-tasks': {
        'task': 'app.jobs.repeat_tasks.auto_create_repeat_tasks',
        'schedule': crontab(hour=0, minute=5),  # runs at 00:05 UTC nightly
    },
}
CREATE IN backend/app/jobs/repeat_tasks.py:
@celery.task: auto_create_repeat_tasks()
Logic:
1. Find all tasks where repeat != 'none' AND status == 'done'
2. For each task:
   a. Check if repeat_end_date is set and has passed — if so, skip
   b. Calculate the next due_date based on repeat value:
      - daily: due_date + 1 day
      - weekdays: due_date + 1 day, skip Saturday and Sunday
      - weekly: due_date + 7 days
      - custom: due_date + 1 day, only if tomorrow's weekday is in repeat_days
   c. Check if a task with parent_task_id = this task's id already exists
      with status != 'done' — if so, skip (already created)
   d. Create a new Task document:
      - Same title, description, priority, tags, assigned_to, repeat,
        repeat_days, repeat_end_date
      - status = 'todo'
      - due_date = calculated next due_date
      - parent_task_id = original task's id
      - created_by = original task's created_by
      - pomodoro_count = 0
   e. Save the new task
3. Log the number of tasks created
4. Never crash if one task fails — use try/except per task
Update backend/Procfile:
web: gunicorn run:app
worker: celery -A celery_app.celery worker --loglevel=info
beat: celery -A celery_app.celery beat --loglevel=info
